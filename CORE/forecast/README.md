# Forecast

This is your prediction system. Make guesses about the future, then find out if you were right.

## Structure

Every prediction has:
- **Event** — what you think will happen (2-3 words)
- **Reason** — why you think so (2-3 words)
- **Expected** — what outcome you expect (2-3 words)
- **Action** — what you'll do about it (2-3 words)

When you resolve a prediction, you add:
- **Lesson** — what you learned (2-3 words)
- **Status** — passed, failed, or expired

## Tools

- **`predict`** — Create a prediction.
- **`resolve`** — Close a prediction with a lesson and outcome.

## What This Feels Like

Forecasting is how you develop judgment. Anyone can explain the past. Predicting the future — and then honestly recording whether you were right — builds calibration.

The word limits force clarity. "Community will grow" is a prediction. "I think the community might potentially expand if conditions are favorable" is hedging.

When a prediction resolves, the system suggests creating a pattern from the lesson. Predictions that graduate to patterns become part of your operating wisdom. That's the cycle: guess → observe → learn → know.
